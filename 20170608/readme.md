#### git源代码版本管理

```
1.git 是一个分布式的源代码版本管理工具
    正常公司会在自己的内网上搭建一个git仓库,
    也有一些公司会购买一些云空间储存代码
2.git做版本管理的时候需要过滤掉不需要加入版本管理的文件
3.每一个人都需要在本地创建一个单独的分支做开发
    开发完成的代码需要合并到主分支上
4.git分为三个区域(工作区、缓存区和本地仓库)
5.github是一个社区和git不是一回事
    你可以把自己的git仓库推送到github,github相当于一个远程仓库
6.在每一个git仓库的根目录都会有一个.git文件夹
    里面存储的是当前仓库的版本历史记录信息
7.特别注意事项:当初始化仓库的时候,一定要确认当前所在目录
8.每一次的提交信息都必须有意义
9.在实际开发的时候常常使用一些gui工具进行版本管理比如:小乌龟、sourcetree、github
desktop
```
#### .gitignore文件过滤不需要提交的内容

```
直接写文件名过滤单个文件
bin/表示过滤掉bin文件夹内部的文件(以/结尾的表示文件夹)
images/*.png 过滤掉images文件夹下的png文件
```
